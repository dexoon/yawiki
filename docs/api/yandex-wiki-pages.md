# Страницы API Яндекс Вики

## Эндпоинты

### GET /v1/pages

Получение информации о странице.

**Параметры запроса:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| slug | string | да* | Адрес страницы |
| id | string | нет | Идентификатор страницы |
| fields | string | нет | Поля для включения в ответ (например, `content`) |

*Параметр `slug` обязателен. API возвращает ошибку валидации, если slug не указан.

> **Важно:** По умолчанию API возвращает только метаданные (`id`, `slug`, `title`, `page_type`). Для получения содержимого страницы добавьте параметр `fields=content`.

**Пример запроса (только метаданные):**

```bash
curl -X GET 'https://api.wiki.yandex.net/v1/pages?slug=mypage' \
  -H 'Authorization: OAuth <токен>' \
  -H 'X-Org-Id: <идентификатор_организации>'
```

**Пример ответа (только метаданные):**

```json
{
  "id": 49362055,
  "slug": "mypage",
  "title": "Заголовок страницы",
  "page_type": "page"
}
```

**Пример запроса (с содержимым):**

```bash
curl -X GET 'https://api.wiki.yandex.net/v1/pages?slug=mypage&fields=content' \
  -H 'Authorization: OAuth <токен>' \
  -H 'X-Org-Id: <идентификатор_организации>'
```

**Пример ответа (с содержимым):**

```json
{
  "id": 49362055,
  "slug": "mypage",
  "title": "Заголовок страницы",
  "page_type": "page",
  "content": "# Заголовок\n\nСодержимое страницы в формате Markdown."
}
```

### POST /v1/pages

Создание новой страницы.

**Параметры тела запроса:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| title | string | да | Заголовок страницы |
| slug | string | да | Адрес страницы |
| page_type | string | да | Тип страницы |
| content | string | нет | Содержимое страницы |
| parent_id | integer | нет | ID родительской страницы |

**Типы страниц (page_type):**

| Значение | Описание |
|----------|----------|
| `page` | Обычная страница |
| `grid` | Динамическая таблица |
| `cloud_page` | Облачная страница |
| `wysiwyg` | Страница с WYSIWYG редактором |
| `template` | Шаблон |

**Пример запроса:**

```bash
curl -X POST 'https://api.wiki.yandex.net/v1/pages' \
  -H 'Authorization: OAuth <токен>' \
  -H 'X-Org-Id: <идентификатор_организации>' \
  -H 'Content-Type: application/json' \
  -d '{
    "title": "Новая страница",
    "slug": "new-page",
    "content": "Содержимое страницы",
    "page_type": "page"
  }'
```

**Пример ответа:**

```json
{
  "id": 49362055,
  "slug": "new-page",
  "title": "Новая страница",
  "page_type": "page"
}
```

## Иерархия страниц и slug

Страницы в Яндекс Вики организованы в иерархическую структуру через slug. Slug содержит полный путь к странице:

```
users/i.sukhorukov/scenarios/analytics/US-AN-001
│     │           │          │
│     │           │          └── страница
│     │           └── подпапка
│     └── папка пользователя
└── корневая папка
```

### Правила формирования slug

1. **Полный путь обязателен** - API требует полный путь, включая префикс `users/`
2. **Папки через слеши** - иерархия реализуется через `/` в slug
3. **Без префикса NOT_FOUND** - запрос без `users/` вернёт ошибку

### Примеры

| URL в браузере | Slug для API |
|----------------|--------------|
| `wiki.yandex.ru/users/i.sukhorukov/mypage` | `users/i.sukhorukov/mypage` |
| `wiki.yandex.ru/users/i.sukhorukov/folder/subpage` | `users/i.sukhorukov/folder/subpage` |

### Создание с parent_id

Страницы также могут иметь иерархию через `parent_id` при создании:

```bash
curl -X POST 'https://api.wiki.yandex.net/v1/pages' \
  -H 'Authorization: OAuth <токен>' \
  -H 'X-Org-Id: <идентификатор_организации>' \
  -H 'Content-Type: application/json' \
  -d '{
    "title": "Дочерняя страница",
    "slug": "child-page",
    "content": "Содержимое",
    "page_type": "page",
    "parent_id": 49362055
  }'
```

**Примечание:** В Яндекс Вики нет отдельного типа "папка" (folder). Папки — это обычные страницы, которые содержат дочерние страницы.

## Доступные эндпоинты (из официальной документации)

Согласно официальной документации, API разделён на секции:
- **Страницы** — операции со страницами
- **Ресурсы страниц** — вложения и ресурсы
- **Динамические таблицы** — операции с таблицами (grid)

## Примечания

- Эндпоинт `/v1/pages/tree` возвращает ошибку `NOT_FOUND` (возможно, не реализован или требует особых параметров)
- Документация по конкретным эндпоинтам страниц недоступна публично (возвращает 403 Forbidden)
